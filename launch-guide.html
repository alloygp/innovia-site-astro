Create the repo from the template
GitHub "Use this template" → clone locally
Manual ▶
Why this step Using "Use this template" creates a clean, unrelated repo — not a fork. This means no shared git history with the starter and no risk of accidentally syncing changes back to the template.
On GitHub
Clone command
git clone https://github.com/alloygp/[clientname-site].git cd [clientname-site]
No Terminal — use GitHub Desktop In GitHub Desktop: File → Clone Repository → GitHub.com tab → find your new repo → choose a local path → Clone. Or in VS Code: open the Command Palette → "Git: Clone" → paste the repo URL.
Install dependencies

Install dependencies
No Terminal — VS Code integrated terminal Open the project folder in VS Code (File → Open Folder). Open the integrated terminal with Ctrl+` (backtick) or Terminal menu → New Terminal, then run npm install. VS Code's terminal is the same thing, just built in — you don't need a separate Terminal app.

Phase 2 — Kickoff Script
3
Run the kickoff script
Generates SEO tracker, launch checker, and pre-fills all config files
Script ▶
Why this step This one command replaces ~20 minutes of tedious manual find-and-replace across config files. It also generates the SEO tracker spreadsheet and the launch readiness HTML — both client deliverables you'd have to make by hand otherwise.
Option A — Ask Claude (recommended for both paths)
Option B — Run it yourself in the terminal
Non-interactive command
python3 scripts/kickoff.py \ --name "Client Name" \ --domain "clientdomain.com" \ --owner "Skyler" \ --phone "(XXX) XXX-XXXX" \ --city "City" \ --state "ST" \ --description "130-160 char meta description" \ --tagline "Short tagline here" \ --notify "notify@clientdomain.com" \ --yes
Tip: If you have the Ahrefs API key set in your shell (export AHREFS_API_KEY=...), backlink data is auto-fetched and baked into the launch checker. Otherwise it'll just be empty.

Phase 3 — Assets & Config
4
Add brand assets
Logo, favicon, and OG image — these must exist before you build
Manual ▶
Why this step The OG image and favicon are referenced in the HTML <head> on every page. If they don't exist, social shares will be broken and the browser tab will show no icon. The logo is loaded by site components — a missing file causes build errors.
5
Fill in any remaining config TODOs
site.ts, email.config.ts, robots.txt, sitemap.xml
Manual ▶
Why this step The kickoff script pre-fills what you gave it. Anything you skipped (Twitter handle, description, etc.) still has a TODO placeholder. These all render into live HTML on every page — launching with TODO values means broken schema and bad meta tags.
src/config/site.ts — the single source of truth
public/robots.txt
